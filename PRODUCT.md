# Product

## Register

product

## Users

- **作者本人（hsjinde）**：日常在手機與電腦上查閱、搜尋、直接在網頁上編輯自己的 Obsidian 筆記；登入後使用編輯回寫與「問資料庫」AI 問答。
- **公開讀者**：透過分享連結或搜尋抵達的訪客，瀏覽白名單資料夾（個人學習／好工具推薦／工作專案）的筆記，像逛一座數位花園。兩種讀者並重：既是自用工具，也是認真經營的對外知識庫。

## Product Purpose

把 GitHub repo `hsjinde/my-note`（Obsidian vault）變成一個永遠同步的公開閱讀網站：push 後幾秒內更新、網頁編輯自動 commit 回 repo、AI 依筆記內容回答問題。成功的樣子：作者把它當成筆記的主要閱讀入口，訪客能舒服地讀完一篇並順著 wikilink 與反向連結探索下一篇。

## Brand Personality

溫暖、安靜、書卷氣。像在書房讀書：紙感底色、襯線標題（Noto Serif TC）、寬鬆行距、慢節奏。介面退後、內容向前；工具功能（搜尋、編輯、AI）存在但不喧嘩。

## Anti-references

- **SaaS 行銷風**：漸層大標、hero 區塊、卡片格陣、行銷式 CTA——這是閱讀工具，不是 landing page。
- 一眼可辨的文件站產生器預設模板感（Quartz／Docusaurus 預設樣式）。

## Design Principles

1. **內容是主角**：每個畫面的視覺重心永遠是筆記正文；chrome（側邊欄、工具列）用低對比與小字級退到背景。
2. **閱讀的舒適優先於資訊密度**：行距、行長、留白向「書」看齊，不向 dashboard 看齊。
3. **連結是導覽的靈魂**：wikilink、反向連結、關聯圖是探索的主要動線，要顯眼、可點、不斷鏈。
4. **工具安靜地在場**：編輯、搜尋、AI 問答隨手可及但不搶戲；未登入訪客看到的是純粹的閱讀體驗。
5. **雙裝置同等公民**：手機查閱與桌機閱讀同樣常見，任何新介面都要在小螢幕上驗證過才算完成。

## Accessibility & Inclusion

盡力而為，不設硬性審查門檻。實務底線：正文對比維持可讀（目標 4.5:1）、動畫尊重 `prefers-reduced-motion`、手機觸控目標不小於 44px。亮／暗雙主題都要維持同等可讀性。
